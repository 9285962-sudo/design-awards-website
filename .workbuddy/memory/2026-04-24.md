## 修复小程序"赛事动态"文章详情页显示空内容问题

### 根本原因（三层叠加）
1. **Cloudflare Pages 对含中文路径的 URL 返回 308 永久重定向**，且 Location 响应头中的中文为乱码（编码错误）。wx.request 虽可跟随 301/302，但 308 + 乱码路径导致无法正确跟随，body 为空（length=0）。
2. `parseArticleHtml` 的正则 `/<article class="article-content">[\s\S]*?<\/article>/` 无法匹配空字符串，content 为空字符串，触发 wxml 的 `wx:else` 分支显示"文章内容加载失败"（用户说的"暂无正文内容"是同一问题）。
3. news.json 的 cover 字段使用相对路径（`/images/...`），小程序无法展示。

### 修复方案
**彻底绕开 Cloudflare 308 问题：把正文 HTML 存入 JSON**

1. **`website/scripts/build-articles.js`** — 修复两处：
   - `markOnlyFirstFeaturedAsFeatured()` 函数 map 时补充 `content: a.content`（该函数重建对象时遗漏了字段）
   - `newsData.push` 和 `strategyData.push` 里加入 `content: article.body`

2. **`mini-program/services/news.js`** — 三处修改：
   - `getNewsList`：给 cover 加 `https://www.52de.cc` 前缀
   - `getStrategyList`：同上
   - `getArticleDetail`：从请求 HTML 改为从 JSON 列表中 find，直接读 content 字段

3. **重新构建并部署**：`node scripts/build-articles.js && npm run build && npx wrangler pages deploy out --branch=main`

### 结论
news.json 和 strategy.json 现在包含完整正文（content 字段），小程序详情页从 JSON 读取，不再依赖 HTML 页面请求。

## 修复文章图片不显示问题

### 根本原因
`build-articles.js` 的 `copyImageToProject` 在源文件（桌面）不存在时直接返回 null，即使 `public/images/articles/` 目录下已有同名文件也不复用。导致重新构建时路径替换失败，content 中的图片 src 仍然是本地绝对路径。

### 修复内容
1. **`build-articles.js`** — `copyImageToProject` 和 `copyRelativeImageToProject`：优先检查目标目录是否已有同名文件，有则直接返回 web 路径。
2. **`mini-program/services/news.js`** — `getArticleDetail`：content 中的 `src="/images/..."` 相对路径替换为 `src="https://www.52de.cc/images/..."` 绝对路径（小程序 rich-text 组件不认相对路径）。

### 结果
14张本地路径图片中13张已恢复（3张因本地和目标都不存在而永久丢失）。
content 里图片路径在 JSON 中存为相对路径，由小程序端运行时替换为绝对路径。

## 修复文章发布问题

### 网站新文章不显示
**根因**：Next.js 静态构建（SSG），首页和策略页通过 `import` 在构建时打包 JSON 数据。只跑了 `build-articles.js` + 部署 public 不够，需要 `npm run build`（生成 out/）再 `wrangler pages deploy out`。

**正确发布流程**：
```
cd website
node scripts/build-articles.js    # 生成 JSON + HTML 文章页
npm run build                     # Next.js 构建（打包最新 JSON 到 out/）
npx wrangler pages deploy out --branch=main   # 部署 out/
```

### 小程序文章图片不显示
**根因**：`article.js` 第127行正则 `/^\/[^\/]/` 把所有以 `/` 开头的相对路径（如 `/images/articles/...`）都当成本地绝对路径跳过了。

**修复**：
- 正则改为 `/^\/(?:mnt|Users|home|tmp)\//i` 只跳过 Unix 绝对路径
- 优化 `article.js`：优先从 JSON 的 `content` 字段读取正文（不再请求 HTML 页面，避免 Cloudflare 308 中文路径问题）

## 奖项详情页统一构建机制

### 问题
- `getStaticProps` 有两套独立逻辑：详细奖（6个）从 award-details.json 取丰富数据，非详细奖（80个）从 awards.json 取基础数据
- 两套逻辑输出的 props 结构不一致，模板需要 `is_detail` 判断
- 导致费用卡片、时间轴等模块在两类奖项间表现不统一

### 解决方案
- 合并两套构建逻辑为**一个统一函数**
- 数据源优先级：detail（丰富）→ award（基础），逐字段 fallback
- 统一字段提取：category、judging_criteria、award_levels、requirements、benefits 等都用同一套逻辑
- 去掉 `is_detail` 标记和对应的模板条件判断
- 所有86个奖项走同一套渲染，有数据就显示，没数据就显示"暂未收录"

### 效果
- 构建速度提升：awards/[id] 从 6951ms 降到 2112ms
- 代码量减少：~60行重复逻辑合并
- 以后新增奖项只需要往 awards.json 加基础数据，有详细数据就写 award-details.json，自动融合

## 小程序数据加载改造：本地 require → 远程 wx.request

### 背景
小程序 `data-loader.js` 原来用 `require('../data/awards.json')` 加载本地 JSON，但 `mini-program/data/` 目录下文件缺失，导致只能加载1个MUSE备用奖。对比页请求 `awards-list.json`（网站不存在该文件）也会404。

### 改造方案（远程加载 + 缓存兜底）
改了5个文件：

1. **`utils/data-loader.js`** — 重写：
   - `require` → `wx.request` 从 `www.52de.cc/data/` 远程加载
   - `fetchWithCache()` 通用函数：先读缓存（24h过期），命中则立即返回+后台静默更新
   - `getAwardsList()` 返回 Promise，从 awards.json 提取16个列表字段
   - `getAwardDetail(awardId)` 返回 Promise，从 award-details.json 按 award_id 查找
   - `filterAwardsByCategory/filterAwardsByFee` 改为接受 awardList 参数

2. **`app.js`** — `loadAwardData` 改为异步：
   - `dataLoader.getAwardsList().then(...)` 替代同步调用
   - 新增 `awardDataReady` 标志位
   - `getAwardDetail` 和 `getWinners` 改为返回 Promise

3. **`pages/awards/awards.js`** — 适配异步：
   - 数据未就绪时轮询（200ms间隔，5秒超时）
   - 增加 `loading` 状态
   - 合并 `filterAwards` 和 `onSearchInput` 的重复筛选逻辑为 `_filterList()`

4. **`pages/award-detail/award-detail.js`** — 改为异步：
   - `loadAwardDetail(awardId)` 异步加载详情
   - 远程无详情时 fallback 到 awards.json 中的基础数据
   - 格式化函数（_formatFeeList 等）提取为独立函数，避免 this 绑定问题

5. **`pages/compare/compare.js`** — 修复404：
   - 原来请求不存在的 `awards-list.json`
   - 改为调用 `dataLoader.getAwardsList()`
