## 昨晚关键问题修复记录（2026-04-19 深夜）

### 问题1：焦点图/文章图片不显示

**现象**：文章详情页图片404，焦点新闻图片无法访问

**根本原因**：
1. 图片路径问题：markdown中使用本地绝对路径（如 `C:\Users\...\图片.jpg`）
2. 部署时序问题：`npm run build` 时图片还未复制到 `public/` 目录
3. 构建脚本问题：`sync-news-data.js` 导致JSON格式错误

**解决方案**：
1. **图片路径标准化**
   - 构建脚本自动提取markdown中的本地图片路径
   - 复制图片到 `public/images/articles/文章名/`
   - 替换markdown中的路径为 `/images/articles/文章名/图片名.jpg`

2. **构建时序修复**
   - 使用 `npm run build-all` 替代分开运行脚本
   - `build-all` 顺序：build-articles.js → next build
   - 确保图片先复制到public，再执行Next.js构建

3. **移除问题脚本**
   - 从 `package.json` 中移除 `sync-news-data.js`
   - 该脚本会导致JSON格式错误（位置210处缺少逗号）

### 问题2：焦点图点击无跳转

**现象**：首页焦点新闻点击后没有跳转到详情页

**原因**：链接路径错误
- 错误：`<Link href={`/news/${id}`}>`
- 正确：`<Link href={`/articles/${id}.html`}>`

**修复位置**：`pages/index.js` 第87行

### 问题3：文章内图片重复显示

**现象**：文章详情页封面图和正文第一张图重复

**原因**：构建脚本自动提取第一张图为封面，但markdown正文里也保留了这张图

**解决方案**：
- 修改markdown，移除正文中的第一张图（因为封面已显示）
- 或在构建脚本中检测封面图和正文第一张是否相同，自动去重

### 问题4：策略文章链接错误

**现象**：参赛策略区域的文章卡片点击无跳转

**原因**：链接路径错误

**修复位置**：`pages/index.js` 第215行

## 关键经验总结

### 图片处理流程
```
用户文章 → 提取本地图片路径 → 复制到public/images/ → 替换为相对路径 → 构建 → 部署
```

### 链接路径规范
- 新闻文章：`/articles/{id}.html`
- 策略文章：`/strategy/articles/{id}.html`

### 构建命令
```bash
npm run build-all && npx wrangler pages deploy out --branch=main
```

### 常见问题排查
1. 图片404 → 检查是否在public目录、路径是否正确
2. 链接无效 → 检查路径是否以.html结尾
3. 页面不更新 → 清除Cloudflare缓存或等CDN刷新

## 策略文章焦点图支持（2026-04-20 下午）

### 问题
策略文章设置了 `featured: true`，但首页焦点图不显示策略文章。

### 根本原因
1. **构建脚本**：策略文章没有提取 `featured` 字段
2. **构建脚本**：策略文章封面图提取逻辑只支持本地路径，不支持网页路径
3. **首页代码**：焦点图区域只显示新闻文章，不包含策略文章

### 修改内容

**1. 构建脚本 `scripts/build-articles.js`**
- 添加 `extractFirstImage()` 函数，支持提取本地路径和网页路径的图片
- 策略文章构建时读取 `featured` 字段
- 支持使用网页路径图片作为封面

**2. 首页 `pages/index.js`**
- 分离焦点策略和普通策略
- 合并新闻和策略的焦点文章，按日期排序
- 焦点图区域同时支持新闻和策略文章
- 根据文章类型生成正确的链接路径

### 数据格式
策略文章现在包含 `featured` 字段：
```json
{
  "id": "文章名",
  "title": "文章标题",
  "cover": "/images/articles/文章名/图片.jpg",
  "featured": true
}
```
