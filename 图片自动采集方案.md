# 设计作品图片自动采集方案

## 问题分析

当前痛点：
- 微信公众号文章图片无法直接抓取（防盗链+动态加载）
- 第三方网站图片采集需要处理各种反爬机制
- 手动保存图片效率低下，无法规模化

## 推荐方案：Playwright 自动化采集

### 方案概述
使用 Playwright 模拟真实浏览器行为，截图保存设计作品图片。

### 技术架构

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   文章URL列表    │────▶│  Playwright脚本   │────▶│  图片存储目录    │
│  (JSON/数据库)   │     │  (截图+下载)      │     │  (按项目分类)   │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                               │
                               ▼
                        ┌──────────────────┐
                        │   图片处理模块    │
                        │ 压缩/水印/格式转换 │
                        └──────────────────┘
```

### 核心功能

1. **智能截图采集**
   - 自动滚动页面加载懒加载图片
   - 识别文章中的设计作品图片区域
   - 支持全屏截图和元素截图两种模式

2. **批量处理**
   - 支持同时处理多个文章URL
   - 自动命名：项目名_序号.jpg
   - 按奖项/年份/项目自动分类存储

3. **图片优化**
   - 自动压缩（WebP格式，质量85%）
   - 添加网站水印
   - 生成多尺寸缩略图

### 文件结构

```
/scripts/
├── image-crawler/
│   ├── index.js              # 主入口
│   ├── crawler.js            # 采集核心
│   ├── image-processor.js    # 图片处理
│   ├── config.js             # 配置
│   └── urls/                 # 待采集URL列表
│       ├── muse-winners.json
│       └── pending-articles.json
└── output/                   # 输出目录
    ├── raw/                  # 原始截图
    ├── processed/            # 处理后图片
    └── thumbs/               # 缩略图
```

### 使用流程

```bash
# 1. 添加待采集文章URL
node scripts/image-crawler/add-url.js \
  --url "https://mp.weixin.qq.com/s/xxx" \
  --project "南沙邮轮码头" \
  --award "BLT建筑设计奖"

# 2. 批量采集
node scripts/image-crawler/index.js --batch-size 5

# 3. 图片自动同步到网站目录
npm run sync-images
```

## 备选方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| **Playwright截图** | 绕过防盗链、可处理动态加载 | 需要浏览器环境、速度较慢 | 微信公众号、动态网站 |
| **直接HTTP下载** | 速度快、资源占用低 | 容易被反爬、无法处理懒加载 | 静态图片资源 |
| **浏览器插件** | 人工干预、精准选择 | 无法自动化、效率低 | 单篇文章采集 |
| **第三方图床API** | 稳定可靠 | 需要付费、有额度限制 | 大规模商业采集 |
