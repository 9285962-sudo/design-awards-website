# 图片采集工具使用指南

## 添加文章URL的方法

### 方法一：直接编辑JSON文件

打开文件：`scripts/image-crawler/urls/pending-articles.json`

在数组中添加新对象：

```json
[
  {
    "url": "https://mp.weixin.qq.com/s/原文链接",
    "project": "项目名称",
    "award": "奖项名称",
    "priority": "high",
    "addedAt": "2026-04-13"
  }
]
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| url | ✅ | 微信公众号文章完整链接 |
| project | ✅ | 项目名称（用于文件名） |
| award | ✅ | 奖项名称（用于分类存储） |
| priority | ❌ | 优先级：high/medium/low |
| addedAt | ❌ | 添加日期 |

### 示例

```json
[
  {
    "url": "https://mp.weixin.qq.com/s/t1QjOqTwUpE2kA2q_2wS5A",
    "project": "南沙国际邮轮码头综合体",
    "award": "BLT建筑设计奖",
    "priority": "high",
    "addedAt": "2026-04-13"
  },
  {
    "url": "https://mp.weixin.qq.com/s/另一个链接",
    "project": "云栖山居度假酒店",
    "award": "WDA世界设计奖",
    "priority": "medium",
    "addedAt": "2026-04-13"
  }
]
```

## 运行采集

### 批量采集（推荐）
```bash
cd scripts/image-crawler
node index.js --file urls/pending-articles.json
```

或双击运行：`crawl.bat --file urls/pending-articles.json`

### 单篇采集
```bash
node index.js --url "文章链接" --project "项目名称" --award "奖项名称"
```

## 采集结果

图片保存在：`output/raw/奖项名称/`

例如：
```
output/
└── raw/
    └── BLT建筑设计奖/
        ├── 南沙国际邮轮码头综合体_01.jpg
        ├── 南沙国际邮轮码头综合体_02.jpg
        └── 南沙国际邮轮码头综合体_03.jpg
```

## 自动化流程（GitHub Actions）

配置完成后，系统会：
1. 每天凌晨2点自动读取 `pending-articles.json`
2. 自动采集所有文章的图片
3. 自动生成文章页面
4. 自动部署到网站

你只需：**添加URL → 提交到GitHub → 自动完成**
