# 设计能网站自动更新方案

## 问题分析

当前痛点：
- 静态HTML页面，数据硬编码在HTML中
- 新增奖项/案例需要手动修改多个页面
- 无法做到"发布即上线"
- 维护成本高，容易出错

---

## 推荐方案：JSON数据驱动 + 自动部署

### 核心思路
将数据与展示分离，数据存储在JSON文件中，页面通过JavaScript动态渲染。

### 实施步骤

#### 1. 数据文件标准化

```
data/
├── awards/
│   ├── awards-index.json      # 奖项索引（首页列表用）
│   ├── muse.json              # MUSE奖完整数据
│   ├── d-ad.json              # D&AD奖完整数据
│   └── ...
└── cases/
    ├── cases-index.json       # 案例索引（列表页用）
    └── case-xxx.json          # 单个案例详情
```

#### 2. 页面改造为动态加载

```javascript
// 首页奖项列表
async function loadAwards() {
  const response = await fetch('data/awards/awards-index.json');
  const data = await response.json();
  
  const container = document.getElementById('awardGrid');
  container.innerHTML = data.awards.map(award => `
    <div class="award-card" data-id="${award.id}">
      <h3 class="award-name">${award.name}</h3>
      ...
    </div>
  `).join('');
}
```

#### 3. 自动部署

**推荐：GitHub + Vercel（免费）**
- 推送代码到GitHub
- Vercel自动检测并部署
- 全球CDN秒级生效

**或：服务器 + Git Webhook**
- 推送触发服务器自动拉取
- 执行部署脚本

### 2. 修改首页加载逻辑

```javascript
// 替换原来的硬编码奖项卡片
async function loadAwards() {
  try {
    const response = await fetch('data/awards-index.json');
    const data = await response.json();
    
    const container = document.getElementById('awardGrid');
    container.innerHTML = data.awards.map(award => `
      <div class="award-card" data-categories="..." data-id="${award.id}">
        <span class="award-badge">${award.category}</span>
        <h3 class="award-name">${award.name}</h3>
        <p class="award-name-en">${award.name_en}</p>
        <div class="award-info">
          <span class="award-info-item">💰 ${award.fee}</span>
          <span class="award-info-item">📅 ${award.deadline}</span>
        </div>
        <p class="award-desc">${award.description}</p>
        <a href="${award.page_url}" class="award-link">查看完整指南 →</a>
      </div>
    `).join('');
  } catch (error) {
    console.error('加载奖项数据失败:', error);
  }
}

// 页面加载时执行
loadAwards();
```

### 3. 新增奖项/案例流程

```
新增奖项：
1. 编辑 data/awards-index.json，添加新奖项对象
2. 创建 data/awards/新奖项.json（详情数据）
3. 提交代码 → 自动部署 → 网站更新

新增案例：
1. 编辑 data/cases-index.json，添加新案例
2. 可选：创建 data/cases/新案例.json（详情）
3. 提交代码 → 自动部署 → 网站更新
```

## 总结

| 方式 | 更新步骤 | 技术门槛 | 维护成本 |
|------|----------|----------|----------|
| **现在** | 改HTML → FTP上传 | 低 | 高 |
| **JSON方案** | 改JSON → Git提交 | 中 | 低 |
| **Notion方案** | Notion里编辑 | 极低 | 极低 |

**建议**：先实施JSON方案，投入1-2天改造，后续更新只需改JSON文件，推送到GitHub即可自动上线。
