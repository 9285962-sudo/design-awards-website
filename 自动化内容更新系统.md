## 方案一：GitHub Actions 自动工作流（推荐）

### 工作原理
利用 GitHub 的免费定时任务，每天自动运行采集脚本

### 配置步骤

**1. 创建 GitHub 仓库**
```
仓库名: design-awards-content
结构:
├── .github/workflows/
│   └── auto-crawl.yml      # 自动工作流配置
├── crawler/
│   ├── index.js            # 采集脚本
│   └── urls.json           # 待采集列表
├── content/                # 采集的内容
│   ├── images/             # 图片
│   └── articles/           # 文章
└── website/                # 网站源码
```

**2. 配置自动工作流** (`.github/workflows/auto-crawl.yml`)
```yaml
name: Auto Content Update

on:
  schedule:
    - cron: '0 2 * * *'    # 每天凌晨2点运行
  workflow_dispatch:        # 支持手动触发

jobs:
  crawl:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install Playwright
        run: |
          npm install playwright
          npx playwright install chromium
      
      - name: Run Crawler
        run: node crawler/index.js
      
      - name: Commit Changes
        run: |
          git config user.name "Auto Bot"
          git config user.email "bot@example.com"
          git add content/
          git commit -m "Auto update: $(date)" || exit 0
          git push
```

**3. 自动部署到网站**
- Vercel/Netlify 自动监听 GitHub 仓库变化
- 内容更新 → 自动触发网站重新部署
- 全程无需人工

## 方案二：自建服务器定时任务

### 适用场景
已有服务器，需要更灵活的控制

### 系统架构

```
服务器定时任务 (crontab)
    ↓
每天凌晨执行采集脚本
    ↓
检测 urls/pending.json 中的新文章
    ↓
自动采集图片 + 提取文字
    ↓
生成网站页面 → 保存到 website/articles/
    ↓
自动提交到 Git → 触发部署
```

### 配置步骤

**1. 创建定时任务脚本** (`scripts/auto-update.sh`)
```bash
#!/bin/bash

# 日志文件
LOG_FILE="/var/log/design-awards/update.log"
echo "[$(date)] 开始自动更新..." >> $LOG_FILE

# 进入项目目录
cd /var/www/design-awards

# 拉取最新URL列表
git pull origin main

# 运行采集
node scripts/image-crawler/index.js --file urls/pending-articles.json

# 生成文章页面
node scripts/generate-articles.js

# 提交更新
git add .
git commit -m "Auto update: $(date +%Y-%m-%d)"
git push origin main

echo "[$(date)] 更新完成" >> $LOG_FILE
```

**2. 添加到 crontab**
```bash
# 每天凌晨3点执行
0 3 * * * /var/www/design-awards/scripts/auto-update.sh
```

## 方案三：智能化 URL 发现（进阶）

### 问题
手动维护 `urls/pending-articles.json` 还是不够智能

### 解决方案：自动发现新文章

**1. 监测目标源**
```javascript
// 配置监测源
const SOURCES = [
  {
    name: '设计腕儿',
    type: 'wechat',           // 微信公众号
    account: 'shejiwaner',
    keywords: ['获奖', '设计奖', 'MUSE', 'BLT']
  },
  {
    name: 'ArchDaily',
    type: 'rss',              // RSS订阅
    url: 'https://www.archdaily.cn/feed/rss.xml',
    keywords: ['award', 'winner']
  },
  {
    name: '谷德设计网',
    type: 'rss',
    url: 'https://www.gooood.cn/feed',
    keywords: ['获奖']
  }
];
```

**2. 自动发现流程**
```
定时抓取 RSS/公众号 → 解析新文章 → 匹配关键词 → 自动添加到待采集队列
```

**3. 智能过滤**
- 已采集过的文章自动跳过（通过URL去重）
- 只采集包含设计作品图片的文章
- 自动识别奖项名称、项目名称
