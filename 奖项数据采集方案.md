# 奖项官网数据采集方案

---

## 一、核心挑战

| 挑战 | 说明 |
|------|------|
| 网站结构差异 | 100个奖项官网，HTML结构各不相同 |
| 数据位置不固定 | 同一字段在不同网站可能位于不同位置 |
| 更新频率不一 | 有的奖项每年更新，有的随时调整 |
| 反爬机制 | 部分网站有反爬虫保护 |
| 数据格式混乱 | 日期格式、货币单位、文本描述不统一 |

---

## 二、解决方案架构

### 整体流程

```
奖项官网列表
    ↓
配置化爬虫规则（每个奖项独立配置）
    ↓
自动化采集引擎
    ↓
数据清洗标准化
    ↓
人工审核校验
    ↓
更新数据库
    ↓
变更通知（邮件/企业微信）
```

---

## 三、技术方案选型

### 方案A：配置化爬虫（推荐）

**核心思想**：为每个奖项编写独立的采集配置，而非独立代码

**配置内容**：
```yaml
# 示例：MUSE Design Awards 采集配置
award_id: muse_design
award_name: MUSE Design Awards
base_url: https://www.museaward.com/

# 页面列表
pages:
  - name: entry_info
    url: /entry-info
    fields:
      - field: entry_fee
        selector: ".fee-table .early-bird"
        type: text
        regex: "\\$(\\d+)"
      - field: deadline_regular
        selector: "#deadline-regular"
        type: date
        format: "MMM dd, yyyy"
      - field: categories
        selector: ".category-list li"
        type: list
```

**优点**：
- 新增奖项只需添加配置，无需写代码
- 字段统一映射到标准格式
- 易于维护和更新

**缺点**：
- 需要为每个奖项单独配置
- 网站改版时需要更新配置

---

### 方案B：AI智能提取（未来方向）

**核心思想**：用大模型自动识别网页中的关键信息

**流程**：
```
抓取网页HTML → 清洗无用标签 → 输入LLM 
→ 提取结构化数据 → 标准化输出
```

**Prompt示例**：
```
从以下网页内容中提取奖项信息，按JSON格式返回：
- 参赛费用（entry_fee）：数字，单位USD
- 常规截止日期（deadline_regular）：ISO日期格式
- 参赛类别（categories）：数组

网页内容：
{html_content}
```

**优点**：
- 无需为每个网站写规则
- 自适应网站改版
- 可处理非结构化文本

**缺点**：
- 成本较高（API调用费用）
- 需要校验准确性
- 响应速度较慢

---

### 方案C：混合方案（最终推荐）

**核心思想**：配置化为主，AI为辅助

```
优先级1：配置化规则（准确、快速、低成本）
    ↓ 规则失效/新网站
优先级2：AI智能提取（兜底方案）
    ↓ 提取结果
人工审核确认
```

## 四、标准化数据字段映射

### 20个标准字段 vs 网页原始字段

| 标准字段 | 常见网页表述 | 提取规则 |
|----------|--------------|----------|
| entry_fee | Entry Fee / Cost / Price / 参赛费用 | 提取数字，统一转USD |
| deadline_regular | Early Bird Deadline / Regular Deadline | 识别日期格式，转ISO 8601 |
| deadline_final | Final Deadline / Last Call | 同上 |
| category_main | Category / Discipline / Field | 映射到标准分类 |
| project_year_limit | Project Completion Date / Eligibility | 提取年限要求 |
| judging_criteria | Judging Criteria / Evaluation | 提取文本 |
| organizer | Organized by / Presented by | 提取机构名 |
| country | Location / Country | 提取国家代码 |

## 五、实施步骤

### Phase 1：基础设施（1-2周）

1. **搭建爬虫框架**
   - Python + Scrapy / Playwright
   - 代理池（防反爬）
   - 定时任务调度

2. **设计配置规范**
   - YAML/JSON配置格式
   - 字段映射规则
   - 验证机制

3. **数据存储**
   - 原始数据表（raw_data）
   - 标准化数据表（awards）
   - 变更历史表（changes）

### Phase 2：核心奖项配置（2-3周）

优先配置高频奖项：
- MUSE Design Awards
- TITAN Property Awards
- IDA Design Awards
- 红点奖（Red Dot）
- iF Design Award
- IDEA Award
- Architizer A+Awards
- WAF World Architecture Festival
- 等等...

每个奖项配置：
1. 分析网页结构
2. 编写CSS/XPath选择器
3. 测试数据提取
4. 验证数据准确性

### Phase 3：自动化运维（持续）

1. **定时采集**
   - 高频奖项：每周一次
   - 中频奖项：每月一次
   - 低频奖项：每季度一次

2. **变更检测**
   - 对比新旧数据
   - 标记变更字段
   - 发送变更通知

3. **异常处理**
   - 网站改版检测
   - 规则失效告警
   - 自动切换到AI提取

## 六、数据质量保证

### 校验机制

| 校验类型 | 说明 |
|----------|------|
| 格式校验 | 日期格式、数字范围、必填字段 |
| 逻辑校验 | 截止日期 > 当前日期、早鸟价 < 常规价 |
| 交叉校验 | 同一奖项不同页面数据一致性 |
| 人工抽检 | 每月随机抽取10%数据人工核对 |

### 数据清洗规则

```python
# 示例：费用标准化
def normalize_fee(fee_text):
    # 提取数字
    number = extract_number(fee_text)
    # 识别货币
    currency = detect_currency(fee_text)
    # 转换为USD
    if currency == 'EUR':
        number = number * 1.08
    elif currency == 'GBP':
        number = number * 1.27
    return round(number, 2)

# 示例：日期标准化
def normalize_date(date_text):
    formats = [
        "%B %d, %Y",      # March 15, 2026
        "%b %d, %Y",      # Mar 15, 2026
        "%d/%m/%Y",       # 15/03/2026
        "%Y-%m-%d",       # 2026-03-15
    ]
    for fmt in formats:
        try:
            dt = datetime.strptime(date_text, fmt)
            return dt.strftime("%Y-%m-%d")
        except:
            continue
    return None
```

## 九、风险与应对

| 风险 | 应对策略 |
|------|----------|
| 网站改版导致规则失效 | 监控+告警+快速更新配置 |
| 反爬封IP | 代理池+请求频率控制+User-Agent轮换 |
| 数据不准确 | 多源校验+人工抽检+用户反馈 |
| 维护成本高 | 优先维护高频奖项，低频奖项降低更新频率 |
